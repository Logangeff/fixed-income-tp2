# FINA 60201A - TP 2

This workspace is organized so we can solve the assignment question by question while keeping the data pipeline reproducible.

## Structure

- `data/raw/`: source files or manually exported datasets.
- `data/processed/`: cleaned datasets generated by the scripts.
- `reports/`: written answers, tables, and future figures.
- `scripts/`: thin entry points to run each question.
- `src/fixed_income_tp2/`: reusable project code.

## Question 1

Run:

```powershell
python scripts/run_question1.py
```

What the script does:

1. Looks for `data/raw/compustat_jnj_debt_quarterly.csv`.
2. Reads the Compustat quarterly debt fields `dlcq` and `dlttq`.
3. Converts the WRDS/Compustat values from millions of USD to USD.
4. Computes the quarterly default point

```text
L_t = DLCQ_t + 0.5 * DLTTQ_t
```

5. Linearly interpolates the daily series from 2024Q4 to 2025Q4.
6. Writes the outputs to `data/raw/` and `data/processed/`.

## WRDS / Compustat Connector

I added a WRDS fetcher so you can pull the assignment data directly from Compustat Fundamentals Quarterly once you have your school login working.

Files:

- `config/wrds.credentials.example.json`: copy this to `config/wrds.credentials.json` and fill in your WRDS username.
- `scripts/fetch_wrds_compustat_q1.py`: fetches Johnson & Johnson quarterly `dlcq` and `dlttq` into `data/raw/compustat_jnj_debt_quarterly.csv`.

Recommended flow:

1. Copy `config/wrds.credentials.example.json` to `config/wrds.credentials.json`.
2. Fill in your `wrds_username`.
3. Run the WRDS fetch script from a Python `3.8` to `3.12` environment with the `wrds` package installed.
4. Re-run `python scripts/run_question1.py`.

Example on Windows, if `py -3.10` is available:

```powershell
py -3.10 -m venv .venv-wrds
.venv-wrds\Scripts\activate
python -m pip install -U pip wheel wrds pandas
python scripts/fetch_wrds_compustat_q1.py
deactivate
python scripts/run_question1.py
```

Why I separated this from the current environment:

- The official WRDS Python package documents support for Python `3.8` through `3.12`.
- This machine is currently on Python `3.14`, so I did not force-install the WRDS package into the main environment.

## Full Part One

The complete Part One workflow is implemented in:

- `scripts/run_part_one.py`
- `src/fixed_income_tp2/part_one.py`

Run it from the WRDS virtual environment:

```powershell
.venv-wrds\Scripts\python scripts/run_part_one.py
```

This script will:

1. Rebuild Question 1 from the Compustat extract.
2. Fetch Question 2 market capitalization from `crsp.dsf_v2`.
3. Fetch the Question 3 one-year FRED zero-coupon series.
4. Fetch the zero-coupon term structure used in Question 5.
5. Estimate end-2025 asset value and asset volatility with the iterative Merton approach.
6. Construct the Question 4 default frontier.
7. Compute the Question 5 Merton credit spread term structure.

## Full Part Two

The complete Part Two workflow is implemented in:

- `scripts/run_part_two.py`
- `src/fixed_income_tp2/part_two.py`

Run it with:

```powershell
.venv-wrds\Scripts\python scripts/run_part_two.py
```

This script will:

1. Download the Nelson-Siegel-Svensson parameter row for `2026-02-27`.
2. Save the assignment AA yield observations for the 11 required maturities.
3. Build the Part Two Question 1 risk-free curve, AA curve, and credit spreads.
4. Calibrate the CIR risk-free term-structure model.
5. Calibrate the Duffee (1999) defaultable-yield model.
6. Price the `5Y`, `4%`, semiannual AA callable corporate bond with a two-factor explicit finite-difference scheme.
7. Compare the callable bond YTM to the matched non-callable bond priced with the closed-form Duffee zero-coupon curve.
